Vincent Pauley

A talented web developer with a passion for Javascript and the growing frameworks/technologies that surround the language.

Front End Web

After teaching myself to code HTML and CSS I landed my first job as a front-end developer at Cornerstone OnDemand in 2014. My responsibilites included slicing PSD files that I or other colleagues had created, and then turning the media into responsive HTML & CSS.

My work on the front end has continued throughout my career as I transitioned into javascript development. I consider Javascript my core programming language and I obsessively continue developing my skillset in this language.

JavaScript

Obviously javascript is part of my font-end skillset, but my diverse experience with the language deserves a bit more explanation. I have been working with javascript since 2012 and have built a strong knowledge of its advanced concepts, including the language's unique take on object-oriented programming.

In addition to my experience with libraies like jQuery & Angular, I have studied the back-end capabilities of the language like NodeJS. As my experience with javascript builds I find that list of programming tasks that cannot be solved with javascript is an ever-shrinking list. Below is a list of some common tasks I solve with javascript on a day-to-day basis.

  • making API calls with Ajax
  • looping through API returns
  • jQuery Animations
  • coding business logic

Design

The oldest of my disiplines is certainly design. I always had a talent for 2D based design (especially drawing/painting), after completing my Art Major and becoming practiced with the Adobe suite I was able to translate my talents into web pages by mastering HTML & CSS. Part of what draws me to programming and web-based design is the maticulous level of control a developer has over their work, a control that is often desired within traditional medias like paint.

My experience with traditional art has come through in my software projects. After countless critiques of my drawings and paintings I gained an ability to step outside of my work anonymously and observe it as though it was my first time seeing it. This gives me a unique and practical approach to page layout.

Back End & DBs

Node JS is my true passion when it comes to the backend. I have used it to build APIs, connect to data bases (both SQL and Mongo), and to create simple apps that access the filesystem. In my career I have had some experience with Java and Perl on the backend, but the dominant portion of my knowledge with back-ends comes from my personal studies with Node.

I have experience working with SQL queries in a production environment, full CRUD operations. But when it comes to databases my preference is Mongo DB. The ability to quickly create document-based collections and shard data across servers is invaluable, not to mention the ability to view data in a JSON format as opposed to the classic SQL table structure.
